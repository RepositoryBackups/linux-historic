分区

您要在何处安装 Red Hat Linux？

首先，您必须为一个或多个磁盘分区定义安装点 ，可能还需要在此时创建和（或）删除分区。

如果您不知道如何对系统分区，请阅读 Red Hat Linux Installation Guide 或 Red Hat Linux Reference Guide 中有关分区的章节。

顶端屏幕将显示现有分区，每个分区均具有以下五个字段：

安装点：

此字段表示在安装和运行 Red Hat Linux 时，分区将安装在何处。如果分区已存在但标记为“not set”，则必须为其定义安装点。您可以使用 “编辑”键或双击此分区。

除非您有理由进行其它分区操作，否则我们建议您创建以下分区：

  • 交换分区（至少 16 MB）-- 交换分区用于支持虚拟内存。如果您的计算机 RAM 等于或小于 16 MB，则必须创建一个交换分区。 即使您的内存大于 16 MB，仍然建议您创建一个交换分区。交换分区大小至少应该等于计算机 RAM 的大小或 16 MB（取两者中的较大值）。

  • /boot 分区（最大为 16 MB）-- /boot 上安装的分区包含操作系统内核以及系统自举进程中使用的文件。由于大多数 PC BIOS 的限制，您最好创建一个较小的分区以保存这些文件。此分区大小不应超过 16 MB。

  • 根分区 (500 MB 至 1 GB) -- “/”（根目录）所驻留的分区。在此分区布局中，所有文件（存储在 /boot 分区中的文件除外）都驻留在根分区上。500 MB 的根分区仅允许相当于工作站类型的安装（可用空间极 少），而 1 GB 的根分区则允许安装所有软件包。

设备：

此字段显示分区的设备名称（例如，hda2 表示主硬盘驱动器上的第二个分区）。

要求：

“要求”字段显示分区的空间大小。如果您要重新定义分区大小，则必须删除当前分区，然后使用 “添加”按钮重新创建分区。

实际：

“实际”字段显示分区中当前占用的空间大小。

类型：

此字段显示分区类型（如 Linux Native 或 DOS）。

在“分区”部分中滚动时，您可能会看到一个或多个分区后以红色文本显示一则“Unallocated Requested Partition”消息。 通常，未分配分区的原因是由于缺少可供分区使用的足够可用空间。

要解决未分配所要求分区的问题，您必须将此分区移至另一个具有可用空间的驱动器，然后重新调整分区大小以适应当前驱动器，或者完全删除此分区。您可以使用 “编辑”按钮或双击分区，以进行以上更改。

对系统进行分区

中间一行按钮用于控制分区工具的操作。您可以在此处添加、编辑和删除分区。此外，还有一些按钮可用于确认您所作的更改、重置和退出分区工具。

添加：

您可以使用“添加”按钮，以请求创建新分区。一旦选定此按钮，屏幕将显示一个对话框，其中包含您必须填写的字段（例如安装点、大小和类型）。

编辑：

您可以使用“编辑”按钮，以更改当前选定分区的安装点。

删除：

您可以使用“删除”按钮，以删除分区。屏幕将显示一个对话框，请求您确认删除操作。

重置：

您可以使用“重置”按钮，以取消所作的任何更改。

创建 RAID 设备：

仅在您具有使用 RAID 的经验时，才可使用“创建 RAID 设备”按钮。要了解更多有关 RAID 的信息，请参阅 Red Hat Linux Reference Guide。

驱动器摘要

此部分用于显示您的计算机磁盘配置。它仅可提供摘要信息，每行包含以下字段：

驱动器：

此字段显示硬盘驱动器的设备名称（例如 hda 或 hdb）。

Geom [C/H/S]：

硬盘驱动器的几何结构显示了磁柱、磁头和扇区数。

合计：

此字段显示硬盘驱动器的合计可用空间。

可用：

您可以在此处查看硬盘驱动器的剩余可用空间。

已用：

这些字段显示当前分配至分区的硬盘空间大小（以兆字节和百分比表示）。
